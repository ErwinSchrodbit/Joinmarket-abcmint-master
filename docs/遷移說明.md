# 遷移說明（JoinMarket → ABCMint RPC）

## 目標
將 JoinMarket 的區塊鏈存取切換為 ABCMint 的 JSON‑RPC，在不改變訂單匹配、隱私與交易流程的前提下完成遷移。

## 差異總覽
- 費用：缺少 `estimatesmartfee/getmempoolinfo`，以 `paytxfee` 為基線，必要時回退到策略預設。
- 證明：不支援 `gettxoutproof/verifytxoutproof`，相關功能降級或停用。
- 錢包：不使用 Core 的 watch‑only 匯入；透過地址過濾與 UTXO 查詢實作視圖。
- RBF：固定為 `False`，交易替代邏輯關閉。

## 設定範例
在 JoinMarket 設定中選擇 ABCMint RPC 為區塊鏈來源：
```
BLOCKCHAIN.blockchain_source = abcmint-rpc
BLOCKCHAIN.rpc_host = 127.0.0.1
BLOCKCHAIN.rpc_port = 8332
BLOCKCHAIN.rpc_user = <user>
BLOCKCHAIN.rpc_password = <password>
```

## 遷移步驟
1. 準備節點：啟動已啟用 JSON‑RPC 的 ABCMint/相容 Bitcoin 節點，確認本機可存取。
2. 更新設定：於 JoinMarket 設定檔中將 `blockchain_source` 設為 `abcmint-rpc`，填入 RPC 鑑權。
3. 執行服務：
   - 後端：`python joinmarket_abcmint/service/start_service.py`
   - 啟動器（推薦）：`joinmarket_abcmint/dist/JoinMarket-ABCMint-LauncherV1.0.0.exe`，詳見 `joinmarket_abcmint/docs/啟動器使用指南.md`
4. 驗證功能：檢查高度/區塊/UTXO 查詢、交易推送與訂單流程是否正常。
